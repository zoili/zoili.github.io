cmake命令行

cmake

生成一个项目构建系统

cmake [<options>] <path-to-source>

使用当前工作目录作为构建目录，使用path-to-source作为源代码目录。指定的路径可以是相对于当前工作目录的绝对路径或相对路径。源树必须包含一个CMakeLists.txt文件，不能包含一个CMakeCache.txt文件，因为后者标识了一个现有的构建目录 。

cmake [<options>] <path-to-existing-build>

使用ath-to-existing-build作为构建目录，并从它的CMakeCache.txt文件中加载到源代码目录的路径，该文件必须已经由以前的CMake运行生成。指定的路径可以是相对于当前工作目录的绝对路径或相对路径

cmake [<options>] -S <path-to-source> -B <path-to-build>

使用 ath-to-build作为构建目录，使用 path-to-source作为源代码目录。指定的路径可以是相对于当前工作目录的绝对路径或相对路径。源树必须包含CMakeLists.txt文件。如果构建树不存在，将自动创建

在生成一个构建系统之后，可以使用相应的本地构建工具来构建项目。例如，在使用Unix Makefiles生成器后，可以直接运行make

Options

讲几个重要的，其他的可以看官网文档

-S <path-to-source>
要构建的CMake项目的根目录的路径

-B <path-to-build>
到CMake将用作构建目录根的目录的路径。
如果目录不存在，CMake将创建它。

-C <initial-cache>
预加载脚本以填充缓存。

当CMake第一次在一个空的构建树中运行时，它会创建一个CMakeCache.txt文件，并为项目填充可定制的设置。此选项可用于指定在第一次通过项目的CMake列表文件之前从其中加载缓存项的文件。加载的条目优先于项目的默认值。给定的文件应该是一个CMake脚本，其中包含使用CACHE选项的set()命令，而不是缓存格式的文件。

脚本中对CMAKE_SOURCE_DIR和CMAKE_BINARY_DIR的引用计算为顶级源代码和构建目录

-D <var>:<type>=<value>, -D <var>=<value>
创建或更新CMake CACHE条目。

当CMake第一次在一个空的构建树中运行时，它会创建一个CMakeCache.txt文件，并为项目填充可定制的设置。此选项可用于指定优先于项目默认值的设置。可以根据需要对任意数量的缓存项重复该选项。

如果给出了:type 部分，则它必须是set()命令文档为其CACHE签名指定的类型之一。如果:type 部分被省略，则如果条目不存在类型，则该条目将创建为没有类型的条目。如果项目中的命令将类型设置为PATH或FILEPATH，则 将被转换为绝对路径。

该选项也可以作为单个参数给出:-D<var>:<type>=<value> or -D<var>=<value>.

需要注意的是，-C和-D参数的顺序非常重要。它们将按照列出的顺序执行，最后一个参数优先于前面的参数。例如，如果你指定-DCMAKE_BUILD_TYPE=Debug，后面跟着一个-C参数，文件调用

那么-C参数将优先，CMAKE_BUILD_TYPE将被设置为Release。但是，如果-D参数出现在-C参数之后，它将被设置为Debug。

如果set(… CACHE …)调用在-C文件中不使用FORCE，并且-D参数设置了相同的变量，-D参数将优先考虑顺序

-U <globbing_expr>
从CMake CACHE中删除匹配的条目。

此选项可用于从CMakeCache.txt文件中删除一个或多个变量，使用*和?都受支持。可以根据需要对任意数量的缓存项重复该选项。

小心使用，你可以让你的CMakeCache.txt无法工作

-G <generator-name>

指定生成系统生成器。

CMake可能在某些平台上支持多个本地构建系统。生成器负责生成特定的构建系统。可能的生成器名称在cmake generators(7)https://cmake.org/cmake/help/latest/manual/cmake-generators.7.html#manual:cmake-generators(7)手册中指定。

如果没有指定，CMake检查CMAKE_GENERATOR环境变量，否则退回到内置的默认选择

大概有如下generator-name：
Borland Makefiles
MSYS Makefiles
MinGW Makefiles
NMake Makefiles
NMake Makefiles JOM
Unix Makefiles
Watcom WMake
Visual Studio 15 2017
Visual Studio 16 2019
Visual Studio 17 2022
…

-T <toolset-spec>
生成器的工具集规范(如果支持的话)。

一些CMake生成器支持工具集规范，以告诉本机构建系统如何选择编译器。详细信息请参阅CMAKE_GENERATOR_TOOLSET变量。

-L[A][H]
列出非高级缓存变量。

列表缓存变量将运行CMake，并从CMake缓存中列出所有未标记为INTERNAL或ADVANCED的变量。这将有效地显示当前的CMake设置，然后可以用-D选项进行更改。改变一些变量可能会导致创建更多的变量。如果指定了A，那么它也会显示高级变量。如果指定了H，它还将显示每个变量的帮助

–log-level=<level>
设置日志 <级别> 。
message()命令将只输出指定日志级别或更高级别的消息。有效的日志级别为ERROR、WARNING、NOTICE、STATUS(默认)、VERBOSE、DEBUG和TRACE。

要使CMake运行之间的日志级别保持不变，请将CMAKE_MESSAGE_LOG_LEVEL设置为缓存变量。如果同时给出了命令行选项和变量，则命令行选项优先。

出于向后兼容性的考虑，——loglevel也可以作为该选项的同义词。

–log-context
启用message()命令输出附加到每个消息的上下文。

此选项只显示当前CMake运行的上下文。要使在所有后续CMake运行中显示上下文持久，请将CMAKE_MESSAGE_CONTEXT_SHOW设置为缓存变量。当给出这个命令行选项时，CMAKE_MESSAGE_CONTEXT_SHOW将被忽略。

–debug-find
将cmake find命令置于调试模式。

在cmake运行过程中打印额外的find调用信息到标准错误。输出是为人类使用而设计的，而不是用于解析。另请参阅CMAKE_FIND_DEBUG_MODE变量，以调试项目更本地的部分

–debug-find-pkg=<pkg>[,…]
在调用find_package( )时，将cmake find命令置于调试模式，其中 是给定的以逗号分隔的区分大小写的包名列表中的一个条目。

类似于——debug-find，但将作用域限制为指定的包

–debug-find-var=<var>[,…]
当使用作为结果变量调用时，将cmake find命令置于调试模式，其中是给定逗号分隔列表中的一个条目。

类似于——debug-find，但将作用域限制为指定的变量名。

–preset <preset>, –preset=<preset>
从 /CMakePresets中读取预设值。和 /CMakeUserPresets.json。预设可以指定生成器和构建目录，以及传递给CMake的变量和其他参数列表。当前工作目录必须包含CMake预设文件。CMake GUI也可以识别CMakePresets。json和CMakeUserPresets。json文件。有关这些文件的详细信息，请参见cmake-presets(7)。

在所有其他命令行选项之前读取预设。预置所指定的选项(变量、生成器等)都可以通过在命令行上手动指定来覆盖。例如，如果预设将一个名为MYVAR的变量设置为1，但用户使用-D参数将其设置为2，则首选值为2。

–list-presets[=<type>]
列出指定type的可用预设。 type的有效值为configure、build、test、package或all。如果省略 ，则假设为configure。当前工作目录必须包含CMake预设文件。

创建一个项目

cmake –build <dir> [<options>] [– <build-tool-options>]
cmake –build –preset <preset> [<options>] [– <build-tool-options>]

–build <dir>
要构建的项目二进制目录。这是必需的(除非指定了预设)，并且必须是第一个

–preset <preset>, –preset=<preset>
使用生成预设来指定生成选项。项目二进制目录是从configurePreset键推断出来的。当前工作目录必须包含CMake预设文件。详细信息请参见预置。

–list-presets
列出可用的生成预设。当前工作目录必须包含CMake预设文件

-j [<jobs>], –parallel [<jobs>]
构建时使用的最大并发进程数。如果省略 ，则使用本机构建工具的默认编号。

如果设置了CMAKE_BUILD_PARALLEL_LEVEL环境变量，则在没有给出此选项时指定默认并行级别。

一些本地构建工具总是并行构建。 值为1的使用可用于限制单个作业。

-t <tgt>…, –target <tgt>…
Build 而不是默认目标。可以给出多个目标，用空格分隔

–config <cfg>
对于多配置工具，请选择configuration

–clean-first
首先构建目标清洁，然后构建 (To clean only, use –target clean.)

-v, –verbose
启用详细输出(如果支持)，包括要执行的构建命令。

如果设置了VERBOSE环境变量或CMAKE_VERBOSE_MAKEFILE缓存变量，则可以省略此选项

–
将剩余选项传递给本地编译工具

安装项目

cmake –install <dir> [<options>]

–install <dir>
要安装的项目二进制目录。这是必需的，必须放在第一位

–config <cfg>
对于多配置生成器，选择configuration

–component <comp>
基于组件的安装。只安装组件

–default-directory-permissions <permissions>
默认目录安装权限。权限格式 <u=rwx,g=rx,o=rx>

–prefix <prefix>
重写安装前缀CMAKE_INSTALL_PREFIX

–strip
安装前先剥离

-v, –verbose
启用详细输出。

如果设置了VERBOSE环境变量，则可以省略此选项

Run cmake –install with no options for quick help.

运行脚本

cmake [-D <var>=<value>]… -P <cmake-script-file> [– <unparsed-options>…]

-D <var>=<value>
定义脚本模式的变量。

-P <cmake-script-file>
将给定的cmake文件作为用cmake语言编写的脚本处理。不执行配置或生成步骤，也不修改缓存。如果使用-D定义变量，则必须在-P参数之前执行此操作
